# BirdCLEF+ 2026 中文运行与提交指南

## 你的 GPU 怎么用

你的机器有 NVIDIA GPU，CUDA Version 12.3，6G 显存。训练模型建议用 GPU，速度会比 CPU 快很多；最终 Kaggle 提交通常按比赛要求在 Kaggle CPU notebook 上推理，所以本地 GPU 主要负责训练权重。

6G 显存建议：

- 模型先用 `tf_efficientnet_b0_ns`，不要一开始上大模型。
- `batch-size` 从 8 开始，不够就降到 4。
- CUDA 可用时脚本默认开启 AMP 混合精度。
- 用 `--grad-accum 2` 或 `--grad-accum 4` 做梯度累积。
- 训练音频长度先用 8 秒，稳定后再试 10 秒。

## 1. 创建环境

```powershell
cd D:\kaggle\birdclef2026
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -U pip
pip install -r requirements.txt
```

如果你之前装过 `kagglehub 1.0.1` 并遇到 `get_web_endpoint` 报错，执行：

```powershell
pip install --force-reinstall kagglehub==0.4.3 kaggle
```

验证 GPU：

```powershell
python scripts/check_gpu.py
```

看到 `cuda available: True` 才说明 PyTorch 能用 GPU。

## 2. 配置 Kaggle 认证

先在 Kaggle 网站加入比赛并接受规则。最稳的下载方式是 Kaggle 官方 API：

1. 打开 Kaggle 右上角头像。
2. 进入 **Settings**。
3. 找到 **API**。
4. 点击 **Create New Token** 或 **Generate New Token**。
5. 下载 `kaggle.json`。
6. 把它放到 Windows 用户目录：

```text
C:\Users\你的用户名\.kaggle\kaggle.json
```

也可以在当前命令行临时设置：

```powershell
set KAGGLE_USERNAME=你的Kaggle用户名
set KAGGLE_KEY=kaggle.json里的key字段
```

注意：不要把完整 token 发到聊天、截图、文档或 Git 仓库里。已经暴露过的话，建议去 Kaggle Settings 里 revoke/regenerate。

## 3. 下载数据

```powershell
python scripts/download_data.py --out data/birdclef-2026 --copy
```

如果 `kagglehub` 认证失败，脚本会尝试用 Kaggle CLI 兜底。CLI 需要安装：

```powershell
pip install kaggle
```

如果依然失败，优先检查三件事：

- 是否已经 Join Competition 并接受规则。
- `kaggle.json` 是否放在 `C:\Users\你的用户名\.kaggle\kaggle.json`。
- 当前虚拟环境是否装了 `kaggle` 和 `kagglehub==0.4.3`。

## 4. 解析数据

```powershell
python scripts/prepare_metadata.py --data-dir data/birdclef-2026 --out-dir data/processed
```

检查生成文件：

- `data/processed/species_list.json`
- `data/processed/train_manifest.csv`
- 如果数据里有声景标签，还会有 `data/processed/soundscape_manifest.csv`

## 5. 用 6G GPU 训练第一版

```powershell
python train.py --data-dir data/birdclef-2026 --meta-dir data/processed --out-dir outputs/exp001 --model tf_efficientnet_b0_ns --epochs 8 --fold 0 --batch-size 32 --grad-accum 1 --duration 8 --channels-last
```

如果显存溢出：

```powershell
python train.py --data-dir data/birdclef-2026 --meta-dir data/processed --out-dir outputs/exp001 --model tf_efficientnet_b0_ns --epochs 8 --fold 0 --batch-size 4 --grad-accum 4 --duration 8
```

如果还溢出，把 `--duration 8` 改成 `--duration 5`。

## 6. 本地生成一次提交文件

```powershell
python infer.py --data-dir data/birdclef-2026 --checkpoint outputs/exp001/fold0_best.pt --out submission.csv
```

多 fold ensemble：

```powershell
python infer.py --data-dir data/birdclef-2026 --checkpoint outputs/exp001/fold0_best.pt --checkpoint outputs/exp001/fold1_best.pt --checkpoint outputs/exp001/fold2_best.pt --out submission.csv
```

本地通常没有隐藏测试集，所以本地 CSV 主要用于检查格式。Kaggle 隐藏测试集只会在提交 notebook 运行时出现。

## 7. Kaggle 网站提交步骤

1. 打开比赛页面并确认已经 Join Competition。
2. 在 Kaggle 上创建 Dataset，上传 `fold*_best.pt` 权重文件。
3. 新建 Kaggle Notebook。
4. 右侧 Add Input 添加 `birdclef-2026` 竞赛数据和你的权重 Dataset。
5. Notebook 设置里关闭 Internet。
6. Accelerator 选择 CPU。
7. 在 notebook 里调用 `infer.py`，输出 `/kaggle/working/submission.csv`。
8. 点击 **Save Version**。
9. 选择 **Save & Run All**。
10. 运行结束后进入版本页面，点击 **Submit to Competition**。

提交前确认：

- `submission.csv` 第一列是 `row_id`。
- 后面的类别列顺序和 `sample_submission.csv` 完全一致。
- 没有使用互联网。
- 推理时间没有超过比赛限制。
